# API capability catalog

Use this as an internal capability map, not a replacement for current endpoint documentation.

## Capability groups

- company, person and relation data
- appointments, tasks and activities
- sales, stages and pipeline data
- projects and project members
- selections, lists and preferences
- documents and archives
- user-defined fields and metadata
- users, roles and access context
- web panels and application navigation
- service, quote and other licensed modules

For each implementation, document exact endpoints, application permissions, tenant prerequisites, pagination, filters, identifiers and write behavior.
