# API capability catalog

Use this as a capability map, not an endpoint specification. Verify exact product names, paths, schemas, coverage and entitlements in current official documentation.

Potential capability groups include company search and information, financial data, management and signatory information, ownership and beneficial ownership, person information, compliance screening, monitoring and change notifications.

For each product initiative, document the customer decision supported, required capability, permitted purpose, country coverage, response fields, freshness expectations, storage rights, failure behaviour and commercial dependency.
