# Documentation

This folder contains the governance, strategy, standards and operating processes for Product OS.

## Main areas

- `company/` — team structure, vision, shared context and AI governance
- `process/` — product lifecycle, workflow and delivery playbook
- `strategy/` — portfolio and architectural strategy that spans products
- `document-owners.md` — accountability and review responsibilities
- `REPOSITORY-ARCHITECTURE.md` — canonical top-level folder model
- `REPOSITORY-AUDIT.md` — verified audit findings and cleanup decisions

## What belongs here

Documents that explain **how the organization makes decisions and operates across projects**.

## What does not belong here

- implementation code: use `apps/` or `platform/`
- reusable vendor or domain reference material: use `10 Knowledge/`
- specialist role prompts: use `Agents/`
- individual project status: use `projects/`

Product-specific documents may remain beside their implementation when they are tightly coupled to that product, such as a product business case or run instructions.
