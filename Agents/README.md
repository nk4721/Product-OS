# Agents

This folder contains the operating instructions for the specialist AI roles in Product OS.

## What belongs here

- role mission and scope
- responsibilities and decision boundaries
- required inputs
- expected outputs and handoffs
- collaboration and escalation rules
- quality, privacy and safety guardrails

## What does not belong here

- project plans or product requirements
- vendor documentation
- source code
- customer-specific records
- company strategy and governance

Those belong in `projects/`, `apps/`, `10 Knowledge/`, or `docs/` as appropriate.

## Operating rule

An agent file defines **who performs work and how the role behaves**. It must link to shared processes and knowledge rather than copying them.

Start with [`product-director.md`](product-director.md) for orchestration and use [`../docs/company/TEAM-STRUCTURE.md`](../docs/company/TEAM-STRUCTURE.md) for the organization model.
