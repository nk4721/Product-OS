# Roaring Knowledge Base

This folder contains Product OS's curated knowledge about Roaring for product discovery, architecture, implementation, testing, commercial planning and support.

It is not a replacement for Roaring's official documentation. Endpoint paths, request and response schemas, authentication requirements, country coverage, product names, rate limits, licences and commercial rights must be verified before implementation.

## Maintenance

Review when:

- Roaring changes its documentation, APIs, authentication or packaging.
- A new country or data domain is introduced.
- A customer contract changes available rights.
- Production incidents reveal undocumented behaviour.
- A product adds a new Roaring dependency.
