# References and review register

## Authoritative sources

Use the current official Roaring developer portal, API reference, product documentation, customer agreement, order form and support communication as the authoritative sources.

This repository intentionally does not reproduce endpoint documentation or large vendor text. Add direct official links here when they have been verified by a maintainer.

## Source hierarchy

1. Signed customer agreement and order form for entitlement and permitted use.
2. Current official API reference for endpoint and schema details.
3. Official implementation guides for authentication and operational behaviour.
4. Official support confirmation for unresolved cases.
5. Controlled sandbox or production observations, clearly marked by environment and date.

## Review checklist

- Confirm official links still work.
- Check authentication and environment changes.
- Check API names, versions and deprecations.
- Check country and data coverage.
- Check rate limits and monitoring behaviour.
- Check contractual storage and use rights.
- Update limitations and affected product documents.

## Evidence rule

When adding a technical claim, record its official source or mark it as an internal assumption. Production observations must not be generalized across customers without verification.
